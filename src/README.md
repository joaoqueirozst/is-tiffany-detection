# is-tiffany-detection
> This project presents a detection and location system for a hexapod robot Tiffany in a Intelligent Space ('is'), using Artificial Neural Networks and a microservices-based architecture.

## Architecture
The detection system receives image streams and publishes them `is-wire` from cameras installed at different points in the laboratory, performing `real-time` inferences to identify the robot's position in the environment.
- Image acquisition from cameras gateway;
- This detection system receives these images, detects them using the **detection service**, and publishes them in a **new topic**.

## Technologies Used
- `YOLOv8` is the neural network model used for real-time object detection. ([YOLO model](https://opencv-tutorial.readthedocs.io/en/latest/yolo/yolo.html));
- [Roboflow](https://docs.ultralytics.com/pt/integrations/roboflow) was used as a tool for `annotating` the hexapod robot and creating `bounding box`, as well as organizing the dataset and applying `preprocessing` operations to robust the system;
- A distributed **microservice** was created for the detection system using `Docker`.

Containerization enables:

- Dependency isolation;
- Reproducible environments;
- Distributed execution;
- Integration with **Intelligent Spaces**.

## Requirements
- Python 3.10+;
- NVIDIA GPU with CUDA support;
- RabbitMQ or AMQP message broker for `is-wire`.

Install Python dependencies using:

```bash
pip install -r requirements.txt
```

## Docker

Pull the Docker Image:
```bash
docker pull joaoqueirozz/tiffany_detector:latest
```

## Service and published topic

The broker connection is configured inside `stream.py`:

```python
broker_uri = 'amqp://guest:guest@10.10.2.211:30000'
```
Change your *IP address*, *port* and *credentials*, according to your infrastructure.

Distributed tracing is configured using:

```python
zipkin_host = 'http://10.10.2.211:9411'
```
The detection system publishes inference results through RabbitMQ topics using the following pattern:

```bash
tiffanyDetector.{camera_id}.Detection
```

Where:

- `camera_id` represents the identifier of the camera responsible for the image acquisition;
- `Detection` represents the robot detection event generated by the microservice.

## Running

Run the service:

```bash
python stream.py
```

or:

```bash
python3 stream.py
```
